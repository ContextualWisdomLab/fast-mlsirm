## 2025-07-08 - HTML Reports Accessibility Enhancements
**Learning:** For static HTML reports containing key-value pairings (like metrics), using semantic `<dl>`, `<dt>`, and `<dd>` elements is much more standard and accessible than `div` with `span` and `strong`. Additionally, purely visual and redundant components (like a compact bar chart alongside the corresponding data table) should be hidden from screen readers using `aria-hidden="true"`. Finally, the default browser styles for `<dd>` usually aren't bold, so CSS must explicitly state `font-weight: bold` if the design intends for the values to be bolded, while `<dt>` requires `font-weight: normal` if previously mapped to a `strong` or if standardizing across browsers.
**Action:** When creating key-value structures, prefer semantic description lists. Hide redundant decorative charts from assistive tech with `aria-hidden`. Always check default browser styles for semantic elements when replacing generic elements like `span` or `strong`.
