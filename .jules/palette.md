## 2025-02-23 - Improve HTML Report Semantic Tags and Accessibility
**Learning:** For key-value data visual displays (like metrics sections in HTML reports), using semantic `<dl>`, `<dt>`, and `<dd>` tags makes the content much clearer to assistive technologies compared to relying on generic `<div>` or `<span>` elements combined with visual styling. Resetting default `<dt>` styling (e.g., setting `font-weight: normal`) prevents cross-browser visual inconsistencies.
Additionally, hiding redundant visual elements (like chart bar tracks) from screen readers using `aria-hidden="true"` declutters the audio reading experience. Using `:focus-visible` instead of `:focus` guarantees that keyboard focus indicators do not confusingly persist after a user clicks on an element with a mouse.
**Action:** When creating or modifying data presentation components, prefer semantic lists (`<dl>`, `<ul>`, `<ol>`) over styled generic containers. Always add `aria-hidden="true"` to purely decorative or redundant visual track elements. Standardize focus outlines to use `:focus-visible` to distinguish between keyboard navigation and pointer interactions.
