## 2024-06-25 - Python 3D Array Broadcasting Bottlenecks
**Learning:** In NumPy, broadcasting arrays to 3D for pair-wise distance or gradient computations (e.g., `params.xi[:, None, :] - params.zeta[None, :, :]`) creates huge O(N*J*D) intermediate matrices that rapidly eat up memory and cause significant slowdowns as dimensions grow.
**Action:** Replace 3D broadcast calculations with vectorized dot products (e.g., `(a - b)^2 = a^2 + b^2 - 2ab`) for distance matrices. For gradients, algebraically rearrange sums to compute values using 2D matrix multiplications instead of creating 3D common term arrays.

## 2025-02-17 - Optimize gradient loops with NumPy matrix masking
**Learning:** In NumPy, replacing python nested loops that operate on sliced arrays with boolean/integer masking and matrix multiplication can yield extreme performance speedups (30x or more). Specifically, when computing `grad_theta[:, d] = (e[:, items] * a[items][None, :]).sum(axis=1)` where `items` are boolean masks for each dimension, creating a full 2D mask `(factors[:, None] == np.arange(D)).astype(float) * a[:, None]` and performing a single dot product `e @ mask` entirely eliminates loop overhead and leverages highly optimized BLAS routines. Similarly, `sum(axis=0)` operations on broadcasted arrays can often be simplified.
**Action:** Always look for opportunities to replace `for`-loops that subset arrays with boolean masks and matrix multiplication or explicit `einsum`/`dot` product operations, even if it requires building an intermediate indicator matrix, as the constant factor overhead in Python loops dwarfs the memory/computation cost of the indicator matrix in NumPy.
